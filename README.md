# Nox Central

Nox Central es el repositorio operativo del ecosistema Nox. Su objetivo es conservar tickets, documentacion, decisiones, proyectos, arquitectura y flujo de trabajo en una unica fuente de verdad.

## Estructura

- `docs/`: documentacion base del ecosistema, arquitectura, agentes, flujo y glosario.
- `tickets/`: gestion de tickets por estado y plantilla oficial.
- `projects/`: seguimiento de proyectos activos y archivados.
- `decisions/`: registro de decisiones importantes.
- `roadmap/`: planificacion y vision temporal.
- `templates/`: plantillas reutilizables para tickets, proyectos y decisiones.

## Flujo de trabajo

Todo trabajo importante debe partir de un ticket aprobado. El flujo base es: idea, diseno, ticket, validacion, implementacion, QA, documentacion y cierre.

## Principio operativo

La documentacion debe ser clara, breve y preparada para crecer sin perder organizacion. Los cambios deben mantenerse pequenos, trazables y alineados con el ecosistema Nox.

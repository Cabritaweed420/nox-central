# Arquitectura

GitHub es la fuente unica de verdad del ecosistema Nox.

## Reglas base

- Toda modificacion importante debe partir de un ticket.
- La documentacion debe reflejar las decisiones relevantes.
- Las decisiones estructurales deben registrarse antes de implementarse.
- Los proyectos deben vincularse con sus tickets asociados.
- La implementacion debe respetar la arquitectura documentada.

## Alcance actual

Este documento define el principio operativo inicial. La arquitectura tecnica futura debera ampliarse mediante tickets y decisiones registradas.
